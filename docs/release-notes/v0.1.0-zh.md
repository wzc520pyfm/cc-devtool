# cc-devtool v0.1.0

## 概述

cc-devtool v0.1.0 是首个公开发布版本。提供零配置的 AI agent 执行调试面板，支持 Claude Code、Cursor、Codex 三大 AI 编程助手的会话解析与可视化，涵盖 skill / rule / MCP / agent 命中追踪、文件读写分析、token 消耗统计、实时监控及可选的透明 API 代理。

**发布日期**：2026-03-23

**规模**：9 commits | 70 files changed | +3,800 insertions

---

## 亮点

- **多 Agent 统一视图**：一个面板同时查看 Claude Code、Cursor、Codex 的所有会话历史
- **执行链路可视化**：时间线、文件影响分析、Agent 拓扑图、数据面板四大视图
- **SQLite 数据补充**：被动读取 Cursor 和 Codex 的本地数据库，补全 JSONL 中缺失的文件操作、模型信息和 token 数据
- **透明 API 代理**：可选启用，捕获 API 流量以补全 token 数据，支持与 cc-switch 链式使用
- **代理控制面板**：在界面中管理代理的启停、配置、统计和数据源切换
- **零配置体验**：安装后直接运行，自动发现本地 agent 数据目录
- **npm 全局安装**：`npm install -g cc-devtool` 即可使用
- **Tauri 桌面应用**：原生桌面应用框架搭建完成（后续版本完善打包）

## 新功能

### 多 Agent 会话解析

支持三种 AI 编程助手的本地数据解析：

- **Claude Code**：解析 `~/.claude/projects/**/*.jsonl`，数据最完整，包含工具调用、token 用量、thinking 内容、MCP 调用
- **Cursor**：解析 `~/.cursor/projects/**/*.jsonl` 和 `.txt` 格式，并从 `ai-code-tracking.db` SQLite 数据库中补充文件操作和模型信息
- **Codex**：解析 `~/.codex/sessions/**/*.jsonl` 和 `index.json`，并从 `state_5.sqlite` 数据库中补充 token 用量和 git 上下文

### 会话时间线

逐轮展示完整的 AI 交互过程：
- 用户消息与 AI 回复
- thinking/reasoning 过程（可折叠）
- 工具调用详情（参数与返回值）
- 自动分类：文件读写、shell 命令、skill、rule、MCP、agent

### 文件影响分析

按文件路径分组展示所有文件操作：
- 读取 / 创建 / 修改 / 删除操作统计
- 从 JSONL 工具调用和 SQLite 数据库双源汇聚

### Agent 执行图

基于 React Flow 的多 agent 层级关系可视化：
- 展示 agent 间调用拓扑
- 节点包含 agent 类型和调用信息

### Dashboard 数据面板

综合数据仪表板：
- 工具使用分布（饼图/柱状图）
- Token 消耗趋势
- Shell 命令执行列表
- Skill / Rule / MCP 命中详情

### 原始文件预览

直接查看 agent 日志源文件：
- JSONL 格式：逐条记录折叠展示，支持类型过滤和关键词搜索
- TXT 格式：语法高亮显示，支持搜索

### 数据完整度指示

每个会话明确标注数据可用性：
- 工具调用 / Token / 文件操作三项独立指示
- `full` / `partial` / `none` 三级状态
- 附带说明原因（如 "Cursor JSONL 不记录 token"）

### 透明 API 代理

可选的 API 流量捕获功能：
- 支持 Anthropic 和 OpenAI 两大 API
- SSE 流式响应透传
- 捕获数据保存至 `~/.cc-devtool/captures/`
- 可与 cc-switch 链式使用：`Agent → cc-devtool → cc-switch → API`

### 代理控制面板

在 Web 界面中管理 API 代理：
- 一键启停
- 端口和上游 URL 配置
- 捕获统计（数量、磁盘占用、最后捕获时间）
- 全局数据源切换（All / Local Only / Proxy Only）
- 配置持久化至 `~/.cc-devtool/config.json`

### CLI 工具

```bash
cc-devtool serve     # 启动面板（默认端口 4173）
cc-devtool proxy     # 仅启动代理（默认端口 4174）
cc-devtool serve --with-proxy  # 面板 + 代理同时启动
```

### 实时监控

- WebSocket 实时推送会话变更
- chokidar 监听 agent 日志文件和 SQLite 数据库变化
- 前端自动刷新会话列表

## 已知限制

- Cursor JSONL 格式不记录工具调用和 token 用量，需通过 SQLite 和代理补充
- Cursor 通过内部代理路由 API 调用，无法被 cc-devtool 代理拦截
- Tauri 桌面应用打包流程尚在完善中
- 暂不支持 macOS 代码签名和 Apple 公证
