## Overview

cc-devtool v0.1.0 is the first public release. It provides a zero-config AI agent execution debugging dashboard supporting Claude Code, Cursor, and Codex session parsing and visualization — covering skill / rule / MCP / agent hit tracking, file operation analysis, token consumption statistics, real-time monitoring, and an optional transparent API proxy.

**Release Date**: 2026-03-23

**Stats**: 9 commits | 70 files changed | +3,800 insertions

## Highlights

- **Unified Multi-Agent View**: Single dashboard for all Claude Code, Cursor, and Codex session history
- **Execution Chain Visualization**: Timeline, File Impact, Agent Graph, and Dashboard — four complementary views
- **SQLite Data Enrichment**: Passively reads Cursor and Codex local databases to supplement missing file operations, model info, and token data from JSONL logs
- **Transparent API Proxy**: Optional API traffic capture for complete token tracking; chainable with cc-switch
- **Proxy Control Panel**: In-app proxy lifecycle management, configuration, statistics, and data source switching
- **Zero-Config Experience**: Install and run — automatically discovers local agent data directories
- **npm Global Install**: `npm install -g cc-devtool` for instant use
- **Tauri Desktop App**: Native desktop app scaffolding complete (packaging in future releases)

## New Features

### Multi-Agent Session Parsing

Supports three AI coding assistants' local data formats:

- **Claude Code**: Parses `~/.claude/projects/**/*.jsonl` — most complete data including tool calls, token usage, thinking content, and MCP calls
- **Cursor**: Parses `~/.cursor/projects/**/*.jsonl` and `.txt` formats, enriched with file operations and model info from `ai-code-tracking.db` SQLite database
- **Codex**: Parses `~/.codex/sessions/**/*.jsonl` and `index.json`, enriched with token usage and git context from `state_5.sqlite` database

### Session Timeline

Turn-by-turn display of the complete AI interaction process:
- User messages and AI responses
- Thinking/reasoning process (collapsible)
- Tool call details (parameters and return values)
- Automatic categorization: file read/write, shell commands, skills, rules, MCP, agents

### File Impact Analysis

File operations grouped by path:
- Read / create / modify / delete operation counts
- Aggregated from both JSONL tool calls and SQLite database sources

### Agent Execution Graph

React Flow-based multi-agent hierarchy visualization:
- Agent call topology display
- Nodes contain agent type and call information

### Dashboard Panel

Comprehensive data dashboard:
- Tool usage distribution (pie/bar charts)
- Token consumption trends
- Shell command execution list
- Skill / Rule / MCP hit details

### Raw File Preview

Direct viewing of agent log source files:
- JSONL format: per-record collapsible display with type filtering and keyword search
- TXT format: syntax-highlighted display with search

### Data Availability Indicators

Explicit per-session data availability markers:
- Independent indicators for tool calls / tokens / file operations
- Three-level status: `full` / `partial` / `none`
- Includes reason descriptions (e.g., "Cursor JSONL does not record tokens")

### Transparent API Proxy

Optional API traffic capture:
- Supports both Anthropic and OpenAI APIs
- SSE streaming response pass-through
- Captured data stored in `~/.cc-devtool/captures/`
- Chainable with cc-switch: `Agent → cc-devtool → cc-switch → API Provider`

### Proxy Control Panel

In-app API proxy management:
- One-click start/stop
- Port and upstream URL configuration
- Capture statistics (count, disk usage, last capture time)
- Global data source switching (All / Local Only / Proxy Only)
- Configuration persisted to `~/.cc-devtool/config.json`

### CLI Tool

```bash
cc-devtool serve     # Start dashboard (default port 4173)
cc-devtool proxy     # Start proxy only (default port 4174)
cc-devtool serve --with-proxy  # Dashboard + proxy together
```

### Real-time Monitoring

- WebSocket real-time session change notifications
- chokidar watches agent log files and SQLite database changes
- Frontend auto-refreshes session list

## Known Limitations

- Cursor JSONL format does not record tool calls or token usage; supplemented via SQLite and proxy
- Cursor routes API calls through its internal proxy, which cannot be intercepted by cc-devtool's proxy
- Tauri desktop app packaging workflow is still being refined
- macOS code signing and Apple notarization are not yet available
