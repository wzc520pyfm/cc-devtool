# cc-devtool v0.1.4

> 会话列表性能优化 + 代理体验升级：cc-switch 一键切换、上游预设、Save & Restart

**[English →](v0.1.4-en.md)**

---

## 概述

本次发布聚焦于 v0.1.3 后用户反馈最集中的两个痛点：

1. 本地会话数量多时，会话列表**加载卡顿**，偶尔甚至**显示「无会话」**，需要切出再切回才能恢复。
2. 切换代理上游（尤其是 cc-switch ↔ 官方直连）操作繁琐。

**发布日期**：2026-04-20

---

## 亮点

- **会话列表显著提速**（尤其 Cursor 会话很多的场景）
- **不再误显示「无会话」** —— 接口失败改为带 Retry 的内嵌提示，已加载的列表保持可见
- **cc-switch 双向 Toggle** —— 一键走 cc-switch，再点一下恢复之前的上游
- **上游预设** —— 任意上游组合都可保存为命名预设，随时一键切换
- **Save & Restart** —— 代理运行中，保存配置 + 热重启一步到位

---

## 主要改动

### 性能 —— 会话列表

- **批量 Cursor 数据补充**：原先每个 Cursor 会话都会对 `ai-code-tracking.db` 发一条 SQL，现在改成每 400 个会话一条 `IN (...)` 查询，大幅减少 SQLite 往返。
- **并发扫描去重**：`GET /api/sessions` 有扫描正在进行时，同时到达的其他请求会共享同一次扫描，避免页面加载 + WebSocket 刷新重复做全盘扫描。
- **按工具做故障隔离**：`listAllSessions` 改用 `Promise.allSettled`。即使某个工具（例如损坏的 Cursor DB）抛错，其他工具的会话依然正常返回，不会整个接口失败。

### 修复 —— 空状态

- **取消过时请求**：客户端在发起新的 `/sessions` 请求前会 abort 上一次请求，避免慢响应覆盖快响应。
- **内嵌错误提示 + Retry**：请求失败不再默默清空列表，而是显示琥珀色提示条（含错误信息与重试按钮），原有列表保持可见。
- **刷新不阻塞**：已有列表时发生后台刷新，只在顶部显示「Refreshing…」字样，列表不会被整屏 Loading 挡住。

### 新增 —— 代理配置

#### cc-switch 双向 Toggle

检测到 cc-switch 时，Configuration 卡片顶部会显示一个开关：

- **ON**：上游路由到 `http://<cc-switch-host>:<port>`
- **OFF**：恢复到你切换前使用的非 cc-switch 上游（没有记录时回退到 `Direct`）

一键切换，不再需要手动编辑两个 URL 字段。

#### 上游预设

新的 Presets 区域：

- **内置 `Direct`** —— `api.anthropic.com` / `api.openai.com`
- **内置 `cc-switch`** —— 检测到 cc-switch 时自动出现
- **用户预设** —— 把任意上游组合保存为命名预设（存储在浏览器 `localStorage`），鼠标悬停可删除。

与当前上游匹配的预设会高亮；当前上游不匹配任何预设时，会出现 `+ Save current` 快捷入口。

#### Save & Restart

代理运行且存在未保存改动时，出现两个按钮：

- **Save** —— 仅保存配置（原有行为）
- **Save & Restart** —— 保存配置并立刻热重启代理，上游变更即时生效，无需手动 stop/start

背后新增 `POST /api/proxy/restart` 接口，原子地执行 stop + start。

## 升级方式

```bash
npm install -g cc-devtool
# 或通过桌面应用 auto-updater 更新
```

所有改动均为增量式，`~/.cc-devtool/config.json` 中的原配置不变。上游预设按浏览器存储在 `localStorage`（键：`cc-devtool:proxy-presets`）。
